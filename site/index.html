What it captures

A structured ledger for how work really happened.

The landing page is static, but the product it describes is built for local capture: terminal commands, agent conversations, Git metadata, and the outputs that explain why a debugging session succeeded or failed.

Terminal events

Record command text, working directory, timestamps, duration, exit status, stdout, stderr, shell, host, and session ID.

AI agents

Tag human work separately from agent activity and keep agent sessions grouped for later analysis.

Git context

Track repository name, branch, commit hash, file changes, commit messages, and diff summaries automatically.

Semantic memory

Privacy first

Useful memory without handing over secrets.

The platform is built local-first, with redaction, exclusion, pause/resume, export, and deletion controls from day one.

  • Local SQLite storage for the MVP.
  • Secret detection for API keys, tokens, passwords, SSH keys, and cloud credentials.
  • Exclude directories and commands you do not want recorded.
  • Pause and resume recording at any time.
  • Export or delete all captured data on demand.
  • Encrypt local storage at rest when the environment supports it cleanly.
